# 项目行为规范

## 设计调整流程

本项目的任何一次设计调整，必须严格按以下顺序执行：

1. 先使用 ui-ux-pro-max 进行设计判断与视觉方向确认。
2. 再使用 gpt-5.3-codex 进行编码实现。
3. 再使用 mini 进行检查。
4. 如果结果未达到 ui-ux-pro-max 标准，继续迭代，直到达到该标准。

## 变更控制规则

- 不要调整用户没有明确要求修改的地方。
- 当用户要求回退时，只回退用户指定的部分，不扩散修改范围。
- 当需求范围很窄时，改动范围也必须保持同样狭窄。

## 生效范围

- 以上规则适用于本项目后续所有设计相关工作，除非用户明确提出新的替代规则。
